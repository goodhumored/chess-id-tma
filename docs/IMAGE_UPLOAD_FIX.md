# Исправление загрузки изображений в событиях

## Проблема
При создании события с изображением возникала ошибка 422, потому что:
1. Фронтенд пытался отправить multipart/form-data напрямую в эндпоинт создания события
2. Кнопки загрузки изображений вызывали сабмит формы
3. Относительные пути к изображениям не могли быть найдены

## Решение

### 1. Создание отдельного эндпоинта для загрузки изображений
- Бэкенд: `/api/v1/events/upload` для загрузки изображений
- Возвращает URL изображения после загрузки

### 2. Обновление компонента ImageUploader
- Создан новый компонент в `src/components/forms/ImageUploader.tsx`
- Использует `httpClient` для загрузки изображений
- Добавлены атрибуты `type="button"` для предотвращения сабмита формы
- Обработка относительных путей к изображениям

### 3. Исправление путей к изображениям
```typescript
// Добавляем baseUrl бэкенда к относительному пути изображения
const imageUrl = response.url.startsWith('http') ? response.url : `${API_CONFIG.BASE_URL}${response.url}`;
```

### 4. Обновление EventForm
- Передача `image_url` в данные события
- Обновлены типы в `ChessEventsRestRepository`

## Файлы, которые были изменены

### Созданы
- `src/components/forms/ImageUploader.tsx` - новый компонент загрузки изображений

### Обновлены
- `src/components/forms/EventForm.tsx` - передача imageUrl в данные события
- `src/infractructure/chess-events-rest.repository.ts` - обновлены типы DTO
- `src/lib/http-client.ts` - поддержка FormData для загрузки файлов
- `src/config/api.config.ts` - конфигурация API
- `.env` - переменные окружения

## Как это работает

1. Пользователь выбирает изображение в форме события
2. Изображение загружается на бэкенд через `/api/v1/events/upload`
3. Бэкенд возвращает относительный путь к изображению
4. Фронтенд добавляет baseUrl бэкенда к пути, получая полный URL
5. URL изображения передается в данные события при создании/обновлении
6. Событие создается с корректным URL изображения

## Возможные проблемы и решения

### Проблема: Изображение не отображается
**Решение**: Проверить, что в `.env` файле указан правильный `NEXT_PUBLIC_API_URL` с протоколом (https://)

### Проблема: Ошибка 422 при создании события
**Решение**: Убедиться, что используется правильный ImageUploader из `src/components/forms/ImageUploader.tsx`

### Проблема: Кнопки загрузки изображений вызывают сабмит формы
**Решение**: Убедиться, что все кнопки имеют атрибут `type="button"`
